# Contributing

Phoxia welcomes contributors with different experience levels, including developers who are still learning.

Before contributing:

1. Read `CODE_OF_CONDUCT.md`.
2. Read `CLA.md`.
3. Open an issue before a large change.
4. Follow repository commands and architecture.
5. Never submit secrets, private data or employer-owned material.

A significant contribution should include the applicable combination of:

- problem and acceptance criteria;
- implementation;
- tests;
- documentation;
- API, event or data-model changes;
- accessibility states;
- failure and recovery analysis;
- release impact.

Submitting a contribution indicates acceptance of `CLA.md`.
