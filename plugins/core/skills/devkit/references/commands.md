# Commands

Use `$phoxia-devkit <command>` in Codex and `/phoxia-core:devkit <command>` in Claude. Users may instead state the same intent naturally in any language; responses follow the user's language while commands stay English.

- `help`
- `status`
- `doctor`
- `project guide|init|adopt|sync|audit`
- `feature start|continue|review`
- `fix start|continue|review`
- `review change|security|reliability|privacy|legal|licensing|accessibility|performance|architecture|full`
- `docs check|adr|feature|rfc|data|operations|changelog|sync`
- `release analyze|prepare|verify`
- `operate readiness|incident|backup|restore|cost`
- `preferences show|set|reset`
- `mode guided|direct`
- `handoff`

`help` lists this surface. `status` summarizes current project state; `doctor` diagnoses it. `project` initializes, adopts, synchronizes, audits or guides. `feature` and `fix` manage scoped delivery. `review` selects evidence-based risk checks. `docs` manages decision and operational records. `release` delegates optional release work. `operate` covers readiness and recovery. `preferences` and `mode` control assistance. `handoff` records durable next state.
