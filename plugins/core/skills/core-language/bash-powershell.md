# Bash and PowerShell

- Quote variables and paths.
- Use strict error handling appropriate to the shell.
- Make scripts idempotent and safe to rerun.
- Validate external input before passing it to commands.
- Avoid `eval` and string-built command lines.
- Use arrays or argument lists rather than shell parsing.
- Provide dry-run behavior for destructive operations.
- Test on the actual target shell and OS.
- Never embed credentials.
