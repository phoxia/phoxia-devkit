# Dart and Flutter

- Use sound null safety and immutable widget configuration.
- Separate UI state from side effects without over-layering.
- Avoid expensive work and allocations in `build`.
- Use isolates only for meaningful CPU work.
- Profile frame time, shader compilation, memory and rebuilds.
- Use sealed classes and exhaustive switches where supported.
- Test widgets, state transitions and accessibility semantics.
- Respect package versions and generated code boundaries.
