# Data, AI and machine learning

- Version data, features, prompts, models and evaluation suites.
- Prevent train/test leakage.
- Record provenance, assumptions and confidence.
- Evaluate against representative and adversarial cases.
- Separate model output from authorization.
- Minimize personal and confidential data.
- Measure quality, latency, cost and drift.
- Make pipelines reproducible and restartable.
- Do not store hidden chain-of-thought as product memory.
