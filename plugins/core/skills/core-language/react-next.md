# React and Next.js

- Follow the installed React and Next.js versions.
- Keep server and client boundaries explicit.
- Avoid effects for derivable state.
- Preserve stable identity only where it matters; do not memoize everything.
- Use exhaustive state modeling for async UI.
- Avoid waterfalls, oversized client bundles and repeated serialization.
- Test accessibility and server/client error states.
- Prefer framework data and cache primitives appropriate to the version.
