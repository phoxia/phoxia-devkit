# Swift and Objective-C

- Use Swift value types and protocols where they clarify ownership.
- Respect actor isolation, `Sendable` and structured concurrency.
- Avoid detached tasks without explicit lifecycle.
- Model state with enums and exhaustive switches.
- Measure with Instruments for CPU, allocation, leaks and energy.
- In Objective-C, use ARC, nullability annotations and explicit ownership.
- Test UI and concurrency on supported OS versions.
- Keep platform API use on the correct thread or actor.
