# Kotlin

- Use null safety, sealed hierarchies and exhaustive `when`.
- Prefer immutable data and expression-oriented code without obscuring control flow.
- Keep coroutine scopes structured and cancellation-aware.
- Avoid `GlobalScope`, blocking calls in suspend code and hidden dispatcher changes.
- Use value classes where they improve domain safety and interop permits.
- Profile allocation and coroutine overhead in real workloads.
- Respect Gradle, Android or server-specific conventions.
