# New language temporary guide

Before applying generic advice:

1. identify compiler or interpreter version;
2. identify official package and build tools;
3. identify formatter, linter, type checker and test runner;
4. identify ownership, error and concurrency model;
5. identify profiler and benchmark tools;
6. inspect repository conventions;
7. verify language claims using primary documentation;
8. write a concise local rule only if the convention will recur.

Do not add a permanent global guide based on a single task.
