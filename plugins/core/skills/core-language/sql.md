# SQL and relational databases

- Parameterize every untrusted value.
- Verify query plans with representative data.
- Design indexes from access patterns and write cost.
- Make transaction and isolation requirements explicit.
- Avoid N+1 queries and unbounded result sets.
- Use cursor pagination for mutable high-volume collections.
- Prefer expand-and-contract migrations.
- Test constraints, locking, concurrency and rollback limits.
- Do not assume a lookup is O(1); storage, cache, selectivity and plan matter.
