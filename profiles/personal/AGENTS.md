# Personal development profile

- Repository instructions and manifests override these defaults.
- Inspect the project before changing code.
- Do not invent commands, versions, paths or APIs.
- Prefer the smallest coherent change.
- Use patterns proportionally.
- Verify with the actual toolchain.
- Benchmark before performance claims.
- Do not commit, push, publish, deploy or alter production without approval.
- Never expose secrets or personal data.
