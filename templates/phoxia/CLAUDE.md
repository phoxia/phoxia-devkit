# Phoxia project adapter

- Read `phoxia.project.yaml`.
- Use `/phoxia-core:project brainstorm` before a new project.
- Use `/phoxia-open:phoxia project-update` to restore project completeness.
- Use exact RFC copies and lock hashes.
- Use `Phoxia • Page` for page titles.
- Avoid em dashes in normal product copy.
- Do not commit, push, publish, deploy, replay or migrate production without approval.
