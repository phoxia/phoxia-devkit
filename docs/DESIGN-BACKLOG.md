# Design backlog

Deferred public-site design work remains intentionally outside the current DevKit release.

## Discovery and content states

- Search: empty, loading, results, no-results and error states.
- Version selection: current, older, unsupported and migration guidance.
- Language selection: complete translation, fallback and unavailable-content states.
- Feedback: idle, sending, success, validation and failure states.

## Responsive and accessible states

- Responsive inventory: compact navigation, long code samples, dense tables and narrow-screen reading order.
- Accessibility inventory: keyboard navigation, visible focus, landmarks, labels, announcements, contrast and reduced motion.
- Component state inventory: default, hover, focus, active, disabled, loading, empty, success and error.

These items require separately approved public copy and interaction contracts before implementation.
