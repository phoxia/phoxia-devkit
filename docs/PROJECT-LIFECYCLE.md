# Project lifecycle

## Brainstorm

Clarify problem, users, value, non-goals, platforms, data, minors, ads, identity, payments, latency, availability, cost, scale, team, budget, license and design.

Output a Project Brief, alternatives, technology recommendation, initial data model, risk register and setup plan.

## Decide

Approve scope, stack, repository strategy, data model, security model and release policy before scaffolding.

## Bootstrap

Create only the accepted setup, CI, tests, documentation, community files and initial schema artifacts.

## Build

Significant capabilities use a Feature Record.

## Synchronize

A local state file records the last audited commit and relevant hashes. Re-run only when relevant files changed.

## Release

Create a release plan, approve the version and publish through GitHub Actions.

## Operate

Maintain telemetry, runbooks, backups, support status and security policy.
